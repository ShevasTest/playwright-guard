# Governance

Playwright Guard is currently maintained by [ShevasTest](https://github.com/ShevasTest).

## Decision making

- Small fixes and documentation improvements are decided through pull-request review.
- Public API changes should begin with an issue and allow reasonable time for community feedback.
- Compatibility, security, and a small maintenance surface take priority over feature count.
- Decisions and their technical reasons should remain visible in issues or pull requests.

## Maintainer responsibilities

Maintainers triage issues, review changes, manage releases, respond to security reports, and enforce the Code of Conduct. A contributor may be invited to become a maintainer after sustained high-quality work, constructive reviews, and demonstrated care for project users.

As the maintainer group grows, release and security permissions will follow least-privilege principles and require review from another maintainer where practical.
